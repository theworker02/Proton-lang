# Code of Conduct

## Our Standard

We want Proton to be a project where people can:

- ask beginner questions without embarrassment
- disagree about design without hostility
- propose ambitious ideas without being mocked
- receive direct technical feedback without personal attacks

## Expected Behavior

- Be respectful and specific
- Assume good intent, then clarify
- Critique code and ideas, not people
- Keep discussions grounded in evidence, examples, and tests

## Unacceptable Behavior

- Harassment or intimidation
- Personal insults
- Deliberate exclusion
- Bad-faith review behavior
- Posting private or security-sensitive information irresponsibly

## Maintainer Guidance

Maintainers should enforce boundaries calmly, consistently, and early. Protecting a healthy working environment matters as much as landing features.
