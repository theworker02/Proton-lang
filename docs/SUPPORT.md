# Support

## Before Opening an Issue

Please gather:

- Proton version
- command you ran
- the `.ptn` snippet or file involved
- compiler/analyzer output
- whether the problem is language semantics, tooling, docs, or editor support

## Best Channels

- Compiler or runtime bug: open an issue with a small reproduction
- Analyzer false positive/negative: include the exact report output
- Docs gap: open a docs issue or PR with the missing section
- Security concern: follow [SECURITY.md](./SECURITY.md)

## Issue Quality Tips

- Prefer one issue per problem
- Include exact expected behavior
- Say whether this is a regression
- If relevant, include `protonc inspect` or `protonc analyze` output

Good reports make Proton much easier to evolve safely.
